# UX Flow Map

This is a first-pass journey map for review planning. Route names are based on
current docs and may need verification during a later UI/UX review.

| Persona or flow | User goal | Expected steps | Copy and next action needs | Failure, empty, pending states | Accessibility and manual test route |
| --- | --- | --- | --- | --- | --- |
| Anonymous visitor | Understand the club and start joining or booking | Visit `/`, read public pages, see any current site notice banners above the header, dismiss a banner with its X button, use contact/join links | Clear club identity, eligibility, support contact, and member login path; banner priority colour and icon convey urgency | Holding page before setup, contact send errors, disabled modules; dismissed banners stay hidden in that browser until the banner is edited | Keyboard nav and heading order on `/`, `/contact`, `/join`; banner dismiss button labelled "Dismiss notice", URGENT banners announced as alerts |
| Applicant | Apply for membership | Open `/join/apply`, submit personal/family/nominator details | Explain nomination, reminder, and admin review sequence | Duplicate application, invalid nominator emails, pending nominations | Form labels/errors on `/join/apply` |
| Nominator | Confirm nomination | Open nomination link, sign in if needed, confirm | Explain who is being nominated and consequence of confirmation | Expired token, wrong account, replaced nominator, already used token | Token route and signed-in confirmation page, to verify |
| New member | Finish first login and setup | Login, change forced password, verify profile/subscription status | Clear next action after approval and setup invite | Expired setup invite, forced password, missing membership invoice | `/login`, `/change-password`, `/dashboard` |
| Existing member booking | Book lodge nights | Choose dates/guests, review quote, handle waitlist or payment | Explain date-only nights, capacity, policies, and cancellation terms | No capacity, invalid stay, expired draft, policy violation | `/book`, `/bookings`, booking detail route |
| Public quote requester | Respond to an emailed lodge quote | Open the secure quote link, review options/price/expiry, then accept, cancel, ask a question, or request changes | Show price, what each option includes, when the quote expires, and the next step after each action | Invalid link, quote no longer active, expired quote (recover by requesting a new one), lodge filled before acceptance (names the full nights; link stays active) | `/booking-requests/respond/[token]`, `/api/booking-requests/respond/[token]` |
| Payment selection | Pay by Stripe or Internet Banking/Xero where available | Choose payment option, confirm payment or invoice instructions | Distinguish card payment from Internet Banking invoice settlement | Payment failed, Xero unavailable, saved card failure, recovery pending | Payment step in booking flow, route to verify |
| Waitlist | Join or accept offered place | Join waitlist, wait for offer, accept before expiry | Explain offer expiry and next action | Offer expired, capacity changed, payment needed | Waitlist UI and emails, route to verify |
| Booking modification | Change dates/guests/promo | Open booking, request or perform allowed edit, settle delta | Explain locked nights, review requirements, delta settlement | Admin review pending, extra payment failed, refund/credit pending | Booking detail modify flow, route to verify |
| Cancellation/refund/credit | Cancel booking and understand settlement | Request cancellation/refund, review policy, see outcome | Explain refund versus member credit and timing | Ineligible cancellation, pending admin review, provider failure | Booking detail cancellation/refund route |
| Profile/family/dependents | Manage personal and family data | Open profile/family pages, edit details, invite/remove dependents | Explain login-capable adult versus dependent behavior | Pending family request, inherited email ambiguity, address autocomplete disabled/unconfigured/unavailable with manual entry still available | `/profile` and family routes, to verify |
| Membership cancellation | Request cancellation for self/family | Start request, collect participant confirmations, wait for admin | Explain account access, booking blockers, rejoin process | Pending participant, rejected, withdrawn, future booking blocker | Member cancellation route, to verify |
| Admin review | Review applications, family groups, cancellations, bookings, refunds | Open admin queues, inspect evidence, refresh/replace pending nominators, approve/reject | Show blockers, irreversible effects, audit trail, next queue item | Empty queue, stale pending, exhausted nomination reminders, pending family-group requests, provider failure | `/admin/dashboard`, relevant queue pages |
| Admin/finance help | Understand what an Admin or Finance page does | Open any Admin or Finance page, click the help icon, read the route-specific popup, then continue the workflow | Help explains common actions, key fields, important notes, and complex page sections without leaving the page | Unknown/nested route falls back to generic Admin or Finance help; dialog can be dismissed without changing form state | `/admin/members`, `/admin/xero/setup`, `/finance`; help button labelled "Open ... help" |
| Admin forms | Save settings and editor changes from long pages or dialogs | Submit member, season, family group, committee, lodge settings, or Xero mapping forms | Bring success feedback back to the top of the admin scroll container, and focus the first error banner when validation fails | Long form remains at submit button while feedback appears above the viewport, dialog body scrolls independently from the page | `/admin/members`, `/admin/seasons`, `/admin/family-groups`, `/admin/committee`, `/admin/setup`, `/admin/xero/setup` |
| Admin settings | Edit shared public footer content | Open `/admin/site-content`, edit the footer blurb, quick links, or affiliations sections, save one section at a time, then review the public footer | Keep footer chrome separate from Page Content page routing and menu settings; show token help for supported footer text tokens | Empty sections, sanitised HTML differences, load or save failure, token fallback for missing Facebook URL | `/admin/site-content`, `/api/admin/site-content`, public website footer |
| Admin settings | Configure seasonal membership type policy | Open `/admin/membership-types`, create/edit/archive/reactivate/reorder types, choose allowed age tiers, configure Membership Type Xero group rules, preview or run seasonal assignment roll-forward | Keep access roles separate from seasonal type policy and committee assignments; explain Age Tier Xero groups for age cohorts, Membership Type Xero groups for status/policy labels, and both when Xero needs both; show roll-forward exceptions before copying | Built-ins present, built-in deletion blocked, empty custom list, duplicate/conflicting Xero rule validation, no prior assignment, inactive type copied from prior season | `/admin/membership-types`, `/api/admin/membership-types`, `/api/admin/membership-types/roll-forward` |
| Admin settings | Configure committee roles and assignments | Open `/admin/committee`, create/edit/archive master roles with role email aliases, review member-linked assignments, and publish contactable entries when appropriate | Keep committee assignment separate from access role and seasonal membership type; new assignments stay hidden until published; role email aliases remain admin/server-only, linked member email is used only as the no-role-email delivery fallback, and phone display is show-phone controlled | No roles, inactive roles, duplicate member-role assignment, hidden/unpublished assignment, non-contactable published assignment, missing role email alias fallback | `/admin/committee`, `/api/admin/committee/roles`, `/api/admin/committee/assignments`, `/api/committee`, `/api/contact` |
| Admin settings | Publish site-wide notice banners | Open `/admin/site-banners`, add or edit a banner with message, Urgent/Warning/Notify priority, and NZ date-only display window, check the live preview, toggle active, delete with confirm | Explain that current banners show above the site header for anonymous visitors and signed-in members, dismissal is per browser, and editing a banner re-shows it to visitors who dismissed it | Empty current/upcoming/past groups, end date before start date, message over 500 characters, inactive banner inside its window | `/admin/site-banners`, `/api/admin/site-banners` |
| Admin member detail | Assign a member's seasonal membership type | Open member detail, compare access role and seasonal type, choose season/type and optional apply-from date, preview affected bookings/subscriptions, enter reason, save | Explain that future booking attempts and reprices use the selected rate/block/subscription policy, existing future bookings are not automatically repriced, raw subscription/Xero history remains visible, and access role remains separate | Missing type, inactive target type, invalid apply-from date, stale preview token, no current assignment, no affected bookings | `/admin/members/[id]`, `/api/admin/members/[id]/seasonal-membership/preview`, `/api/admin/members/[id]/seasonal-membership` |
| Admin member detail | Assign committee roles | Open member detail, add or edit committee assignment, choose role, set blurb/sort/published/show-phone/contactable flags, or deactivate assignment | Show committee assignment beside access role and seasonal type without changing either | No active roles, existing inactive assignment, duplicate role assignment, hidden by default | `/admin/members/[id]`, `/api/admin/committee/roles`, `/api/admin/committee/assignments` |
| Admin recovery visibility | Find stuck states across domains | Open `/admin/stuck-states`, triage by severity/owner, follow target link | Keep payment, Xero, email, waitlist, bed-allocation, and lodge signals in one scan | Empty queue, disabled modules, provider queue failures, stale recovery states | `/admin/stuck-states`, `/api/admin/stuck-states` |
| Finance/Xero | Monitor accounting and reconcile provider state | Open finance/admin Xero pages, run safe syncs, review failures | Explain the shared operational Xero connection used by finance reports | Missing credentials, failed sync, wrong access level | `/finance`, `/admin/xero`, guarded role checks |
| Lodge/kiosk | Manage arrivals, departures, chores, lodge info | Open `/lodge`, sign in or PIN, update day operations | Show current date, accountable actor, safe sign-out | No access, stale PIN session, empty roster | `/lodge`, keyboard/touch target checks |

## Review Prompts

For every journey, verify the next action, failure state, pending state,
accessible form semantics, and destructive/financial-action explanation.
