Where we've landed

  • One screen per lodge; rooms down the side, three nights across.
  • Everyday view: the bar board (names up to 5).
  • Whole-lodge: blockout approved; the rotating welcome for a fully-out group.
  • All-singles full house: by-booking, Room | Guest.
  • Names as first name + surname initial; groups keep name + headcount.
  • Custodian in the footer (only when a lodge has one).

Still open

  • How the display switches between the room-row board and the by-booking board as a night fragments (auto, or fixed).
  • Dark, daylight, or auto by time of day.
  • Wiring the static-core / cycling-corner (housekeeping → checkout → notice) into the chosen everyday board.
  • Whether to visually link a group's non-consecutive room bars.
Versioning: a new iteration is saved as a new file and added here; the previous file is never overwritten. Review-approved whole-lodge designs are frozen in lobby-display/approved/. Files live in Alpine/lobby-display/ — open this index.html from that folder and each “Open” link goes to the saved page; the “shareable” link is a hosted copy. Not in the app repo. Mock data throughout — the real display is data-driven from the club site.
